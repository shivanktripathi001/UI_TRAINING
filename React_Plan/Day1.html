TOPIC 01

What is React & Why React?

React is a JavaScript library built by Facebook (Meta) in 2013 for building User Interfaces. It lets you build complex UIs by breaking them into small, reusable pieces called Components.

Think of React like LEGO blocks. Instead of writing one giant HTML file, you build small components (Header, Button, Card) and assemble them together.

üß©

Component-Based

Build encapsulated pieces that manage their own state

‚ö°

Virtual DOM

Only updates what changed ‚Äî super fast rendering

üîÑ

Declarative

You describe the UI, React figures out the updates

üåç

Huge Ecosystem

Millions of developers, libraries, and jobs worldwide

üí° Real-World Analogy

Imagine you're building a dashboard. Without React, changing a user's name means rewriting the whole page. With React, only the Name component re-renders ‚Äî everything else stays untouched. This is the power of the Virtual DOM.

React Virtual DOM

<App>
  <Header />
  <Name /> ‚Üê changed
  <Footer />
</App>
‚Üí

Real DOM

<div id="app">
  <header>...</header>
  <span>John</span> ‚Üê only this!
  <footer>...</footer>
</div>

Without React (Vanilla JS)

  • Manual DOM manipulation
  • Hard to maintain large apps
  • State management is painful
  • Code becomes spaghetti fast

With React

  • Declarative, readable code
  • Reusable components
  • Built-in state management
  • Scales beautifully

There are two main ways to create a React app. In 2024+, Vite is the recommended choice ‚Äî it's blazing fast. CRA (Create React App) is the older method.

‚ö° Vite (Recommended)

  • Super fast dev server
  • Instant Hot Module Reload
  • Modern tooling (ESM)

üê¢ Create React App

‚ö†Ô∏è Prerequisites

TOPIC 03

JSX ‚Äî Syntax & Rules

JSX stands for JavaScript XML. It lets you write HTML-like syntax directly inside JavaScript. It looks like HTML, but it IS JavaScript under the hood.

ü§î What JSX Actually Does

JSX is just syntactic sugar. Babel converts it to plain JavaScript React.createElement() calls. You never have to write those manually ‚Äî JSX handles it!

JSX (what you write) 
const element = (
  <h1 className="title">
    Hello World!
  </h1>
);
What Babel compiles to 
const element =
  React.createElement(
    "h1",
    { className: "title" },
    "Hello World!"
  );

‚ö° The 6 Golden Rules of JSX

Rule 1: One Root Element

JSX must return a single parent element. Wrap multiple elements in a <div> or React Fragment <></>

Rule 2: className, not class

HTML's class is a reserved word in JS. In JSX, use className instead.

Rule 3: Self-close Tags

Tags without children MUST be self-closed: <img /> <input /> <br />

Rule 4: camelCase Attributes

onclick ‚Üí onClick, onchange ‚Üí onChange, tabindex ‚Üí tabIndex

Rule 5: JS in Curly Braces { }

Embed any JS expression with { }. You can use variables, math, function calls.

Rule 6: for ‚Üí htmlFor

The for attribute in labels is htmlFor in JSX because for is a JS keyword.

üõ†Ô∏è Practical JSX Examples

JSX Rules ‚Äî Practical Example 
// ‚úÖ CORRECT JSX
function MyComponent() {
  const name = "Rahul";
  const age = 22;
  const isLoggedIn = true;

  return (
    {/* Rule 1: Single root ‚Äî use Fragment */}
    <>
      {/* Rule 2: className not class */}
      <h1 className="title">
        {/* Rule 5: JS expressions in curly braces */}
        Hello, {name}! You are {age} years old.
      </h1>

      {/* Rule 5: Conditional rendering */}
      {isLoggedIn && <p>Welcome back!</p>}

      {/* Rule 3: Self-closing tags */}
      <img src="avatar.png" alt="user" />
      <input type="text" />

      {/* Rule 6: htmlFor instead of for */}
      <label htmlFor="email">Email</label>

      {/* Rule 4: camelCase events */}
      <button onClick={() => alert("clicked!")}>
        Click Me
      </button>

      {/* Math expressions work too! */}
      <p>Next year: {age + 1}</p>
    </>
  );
}

‚ùå Common JSX Mistakes to Avoid

Wrong vs Right 
// ‚ùå WRONG ‚Äî two root elements
return (
  <h1>Title</h1>
  <p>Text</p>   // Error!
);

// ‚úÖ RIGHT ‚Äî wrap in Fragment
return (
  <>
    <h1>Title</h1>
    <p>Text</p>
  </>
);

// ‚ùå WRONG ‚Äî class instead of className
<div class="box"></div>

// ‚úÖ RIGHT
<div className="box"></div>

// ‚ùå WRONG ‚Äî img not self-closed
<img src="pic.jpg">

// ‚úÖ RIGHT
<img src="pic.jpg" />
TOPIC 04

Rendering Elements to the DOM

Rendering is how React takes your components and puts them onto the actual webpage. The entry point is main.jsx (or index.js in CRA).

üîë The Key Concept

React renders everything into one single HTML element ‚Äî <div id="root"> in your index.html. React takes over this div and renders your entire app inside it.

Step 1 ‚Äî The index.html (the shell)

index.html 
<!DOCTYPE html>
<html lang="en">
  <body>
    <!-- React mounts your entire app here -->
    <div id="root"></div>

    <!-- Vite loads main.jsx as the JS entry -->
    <script type="module" src="/src/main.jsx"></script>
  </body>
</html>

Step 2 ‚Äî main.jsx (the entry point)

src/main.jsx 
import React from 'react'
import ReactDOM from 'react-dom/client'
import App from './App.jsx'

// 1. Find the root div in index.html
const rootElement = document.getElementById('root')

// 2. Create a React root
const root = ReactDOM.createRoot(rootElement)

// 3. Render your App component into it
root.render(
  <React.StrictMode>
    <App />
  </React.StrictMode>
)

Step 3 ‚Äî Your first App.jsx component

src/App.jsx ‚Äî Your First Component 
// A React Component is just a function that returns JSX!

function App() {
  const user = "Rahul";
  const currentYear = new Date().getFullYear();

  return (
    <div className="app">
      <h1>Hello, {user}! üëã</h1>
      <p>Welcome to React in {currentYear}</p>
      <button onClick={() => alert("React is awesome!")}>
        Click Me!
      </button>
    </div>
  );
}

export default App; // Must export to use in other files

‚úÖ What StrictMode Does

<React.StrictMode> is a development helper. It double-invokes your functions to catch bugs early and warns you about deprecated API usage. It only runs in development ‚Äî no impact on production builds!

TOPIC 05

React DevTools Setup

React DevTools is a browser extension that lets you inspect your React component tree, view props and state, and debug your app like a pro. It's an absolute must-have.

Install the Extension

Go to your browser's extension store and search for "React Developer Tools" by Meta. Available for Chrome, Firefox, and Edge.

Quick Links

üîµ Chrome: Chrome Web Store ‚Üí Search "React Developer Tools"
üü† Firefox: Firefox Add-ons ‚Üí Search "React Developer Tools"

Open your React app in the browser

Run npm run dev and open http://localhost:5173. The React DevTools icon in the toolbar turns blue/active when a React app is detected.

Open DevTools and find the React tabs

Press F12 (or Cmd+Option+I on Mac) to open DevTools. You'll see two new tabs: ‚öõ Components and ‚öõ Profiler

üß©

Components Tab

Shows the entire component tree of your app. Click on any component to see its props and state. You can even edit values in real-time!

üìä

Profiler Tab

Records your app's performance. It shows which components re-rendered, why, and how long they took. Essential for optimizing slow apps.

üîç

Search Components

Press Ctrl+F inside the Components panel to search for any component by name ‚Äî super useful in large apps.

‚ö°

Inspect on Page

Right-click any element on the page ‚Üí "Inspect" ‚Üí then click the arrow icon in DevTools to jump to that element's React component instantly.
